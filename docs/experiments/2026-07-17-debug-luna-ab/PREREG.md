# Debug-lever A/B on DeepSWE-format sympy real bugs — PRE-REGISTRATION (2026-07-17)

Written BEFORE any model arm runs. Goal: find a supergoal edit that raises debugging score
(resolved) vs BOTH no-skill and current v0.9.0, and prove it. Prior history: 6 null results on
debug-skill levers; 2026-07-17 autopsy showed the one claimed "debug loss" was sampling noise.
This experiment therefore pre-commits to seeds, decision rules, and kill rules up front.

## Rig

- Tasks: 6 custom DeepSWE-format tasks converted from validated SWE-bench_Lite sympy real bugs
  (provenance: `docs/experiments/2026-07-04-swt-assertflip-realbug-ab/instances*/`). All are
  symptom-only bug reports (no fix diffs in the problem statement — telegraphed instances 22005,
  24152 excluded by construction):
  - sympy-21627 cosh `is_zero` RecursionError
  - sympy-20442 `convert_to` combines orthogonal units
  - sympy-21055 `refine()` fails on complex-argument simplification
  - sympy-23191 vector pretty_print display-order bug
  - sympy-24909 milli prefix `milli*W == 1` bug
  - sympy-22714 `evaluate(False)` Point2D imaginary-coordinates crash
- Format: standard DeepSWE v1.1 task (task.toml, environment image, hidden tests/, oracle
  solution/, shared verifier grader with junit node-id whitelists from the instance F2P/P2P sets).
  Environment mirrors the locally validated rig: python 3.12 + mpmath==1.3.0, sympy at
  base_commit, PYTHONPATH=/app, no internet at runtime.
- Runner: `templates/harness-eval-external/deepswe/run-full-cycle.mjs` → pier → Docker.
  Agent codex, model `gpt-5.6-luna`, reasoning effort **medium**, declared agent budget 900 s,
  stop policy = runner default (no manual interrupt; `runner_timeout` = invalid arm).
- Benchmark root: `/tmp/deep-swe-sg` (pinned ref) + these 6 untracked task dirs generated by
  `gen_tasks.py` in this directory.

## Validation gate (before any model run)

Every task must pass BOTH, via pier's built-in agents (zero model tokens):
1. `--agent oracle` (gold patch): reward = 1 (all F2P pass, no P2P failure).
2. `--agent nop` (no change): reward = 0 with ALL F2P failing and ALL P2P passing.
Tasks failing either gate are fixed or replaced from the validated backup pool
(20212, 21612, 21847, 23262, 24066, 24102, 24213) before iteration 0. (skrub segfault lesson.)

## Arms

- `baseline`: stock task, no skill.
- `v090`: current shipped supergoal (dev-v2 `350eb96`) embedded by `prepare-supergoal-arm.mjs`.
- `candN`: v0.9.0 + one lever (designed from iteration-0 failure autopsy; max 3 candidates).

## Metrics

- Primary: per-run binary `reward` (resolved: all F2P pass AND no P2P regression).
- Secondary: f2p fraction (partial), p2p regressions, agent time, tokens, patch size.

## Design and decision rules (pre-committed)

1. **Iteration 0 — ceiling screen then baseline.** 1 seed × 6 tasks × {baseline, v090}.
   Drop tasks where both arms hit reward=1 (no headroom) or where the verifier is invalid.
   If <4 tasks survive, pull replacements from the backup pool (one re-roll). Then complete
   surviving tasks to 3 seeds/arm. Autopsy every failure per-test (2026-07-17 termenv method)
   before designing any lever.
2. **Screening (per candidate):** 3 seeds × surviving tasks. Advance only if
   resolved_sum(cand) > resolved_sum(baseline) AND > resolved_sum(v090), and no new p2p
   regressions.
3. **Confirmatory (fresh seeds):** 3 additional seeds on the same tasks, decided ONLY on the
   pooled fresh-seed data: one-sided stratified permutation test (permute arm label within
   task×seed cell) on resolved, p < 0.05 vs EACH baseline. No optional stopping; the
   confirmatory n is fixed before launch. Screening data may not be pooled into the
   confirmatory test.
4. **Guardrail:** the surviving candidate runs csstree-shorthand-expansion-compression
   (feature-add) × 2 seeds; must not fall below v0.9.0's known result band (79/79 at n=1;
   guardrail fails only on clear regression, f2p < 78).
5. **Equal-compute control:** if the winning lever adds turns/passes, run a naive
   equal-compute control (v0.9.0 + "take another verification pass") on the same seeds;
   the lever must beat the control or the claim is "compute, not method".
6. **Kill rules:** all 3 candidates fail screening → report the 7th null honestly. Both
   baseline arms ≥ 5/6 resolved at iteration 0 across seeds → rig has no headroom at
   luna-medium; stop and report rather than fish.

## Interpretation limits

n=3+3 seeds × ≤6 tasks is a directional-to-moderate evidence tier. A p<0.05 confirmatory win
here is "proven on this rig at this model/effort", not a universal claim. All arms share model,
effort, timeout, benchmark ref, and environment images; only the embedded skill text differs.
